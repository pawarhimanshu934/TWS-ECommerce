When working with AWS EKS Cluster :

1) #when using EKS cluster for your project, ensure you have installed the required csi driver for AWS EBS storage provider. You can install it using the console. Go to Cluster -> Add ons -> Select AWS EBS CSI Driver -> Install it

#your pvc won't be bounded to pv until the csi driver is installed. it will be in pending state.

2) use 'gp2' storage class for AWS EKS, if using Kind cluster, you can use 'standard' storage class

3) Instaling Mertics Server is important, HPA and VPA depeneded on it.

4) When working with API Gateway ( Ingress replacement ), install necessary CRUDS. Refer GatewayAPI with kgateway.md

5) Once your EKS is setup on AWS, you can connect to EKS cluster from your local using Kubectl.
   Ensure kubectl is installed on your local
 
   Command : aws eks update-kubeconfig --region <region-code> --name <cluster-name>

6) Refer ArgoCD doc for setting it up.
