# AWS EKS Cluster Setup Guide

When working with AWS EKS Cluster:

## 1. Install AWS EBS CSI Driver

When using EKS cluster for your project, ensure you have installed the required CSI driver for AWS EBS storage provider. You can install it using the console:
- Go to Cluster → Add-ons → Select AWS EBS CSI Driver

**Note:** Your PVC won't be bounded to PV until the CSI driver is installed. It will be in pending state.

## 2. Storage Class Selection

- Use `gp2` storage class for AWS EKS
- If using Kind cluster, use `standard` storage class

## 3. Install Metrics Server

Installing Metrics Server is important as HPA and VPA depend on it.

## 4. API Gateway Setup

When working with API Gateway (Ingress replacement), install necessary CRDs. Refer to [GatewayAPI with kgateway.md](./GatewayAPI%20with%20kgateway.md)

## 5. Connect to EKS Cluster from Local

Once your EKS is set up on AWS, you can connect to EKS cluster from your local using kubectl.

**Prerequisites:** Ensure kubectl is installed on your local machine

**Command:**
```bash
aws eks update-kubeconfig --region <region-code> --name <cluster-name>
```

## 6. ArgoCD Setup

Refer to the ArgoCD documentation for setting it up.
